The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). The project adheres to [Semantic Versioning](https://semver.org/) starting at `1.0.0`.

## [1.0.0] — 2026-05-13

### Overview

First stable release of the `@dokhna-tach/zatca` family. This is the initial public, citable, semver-stable surface for Saudi Arabia ZATCA Phase 2 e-invoicing in Node.js. Four packages ship in lockstep at `1.0.0`:

| Package | Purpose |
|---------|---------|
| `@dokhna-tach/zatca` | Core: XML build, XMLDSig signing, ZATCA SHA-256 hashing, TLV QR, API client, onboarding |
| `@dokhna-tach/zatca-storage-memory` | In-memory `StorageAdapter` for tests/dev |
| `@dokhna-tach/zatca-storage-mongo` | MongoDB (Mongoose) `StorageAdapter` |
| `@dokhna-tach/zatca-storage-postgres` | PostgreSQL (raw `pg`) `StorageAdapter` |

### Public API

**Invoice builders (6):**
- `buildStandardTaxInvoice`
- `buildStandardCreditNote`
- `buildStandardDebitNote`
- `buildSimplifiedTaxInvoice`
- `buildSimplifiedCreditNote`
- `buildPhase1Invoice`
- `buildPhase1CreditNote`

**Crypto / XML / QR:**
- `signInvoiceXml`, `signedPropertiesHash`, `invoiceHash` (ZATCA SHA-256, byte-identical to ZATCA reference)
- `buildQR` (TLV, base64-encoded)
- `generateKeyPair`, `generateCsr`, `probeOpenSSL`
- 3 golden vectors with byte-identity assertions: standard tax invoice, simplified tax invoice, credit note

**API client (ZATCA endpoints):**
- `issueComplianceCertificate`
- `runComplianceTests`
- `issueProductionCsids`
- `clearInvoice`, `reportInvoice`
- `checkInvoiceStatus`, `cancelInvoice`
- Native `fetch` with configurable retry, exponential backoff, and timeout
- Sandbox / simulation / production environment URLs baked in

**Onboarding:**
- `onboard()` — full end-to-end orchestration: generate keys → CSR → compliance CSID → run compliance tests → production CSIDs

**Storage:**
- `StorageAdapter` contract with conformance suite shared across all three adapters
- 13 conformance tests pass on memory, mongo, and postgres

**Validation:**
- Zod schemas for every public payload
- CRN, VAT, UUID, and address validators

### Quality bar

- 332 tests passing across all packages (50 test files)
- Strict TypeScript (`noImplicitAny`, `strictNullChecks`, no `any` in source)
- Zero `console.*` in published code
- Dual ESM + CJS bundles, TypeScript declarations for both
- Node 20 and Node 22 supported (CI matrix: Ubuntu + macOS)
- `pnpm audit` clean of high/critical
- All transitive runtime dependencies are MIT / Apache-2.0 / BSD / ISC (no copyleft)

### Breaking changes

None — this is the first stable release. Pre-`1.0.0` was not published.

### Known gaps for v1.1.0

- BSL 1.1 full legal text body is still abbreviated in `LICENSE` (parameter block is authoritative; full upstream body must be pasted from <https://mariadb.com/bsl11/> before publishing).
- `SECURITY.md`, `CODE_OF_CONDUCT.md` use placeholder `@dokhna-tach.example` emails — must be replaced with real maintainer contacts.
- `@fidm/x509` is unmaintained; v1.1.0 will replace it with `pkijs` (already a transitive dep).
- OpenSSL CLI binary is currently a runtime dependency for CSR generation; a pure-JS path is on the v1.1.0 roadmap.
- Optional PDF/A-3 attachment sub-package is on the v1.1.0 roadmap.
- 3 moderate `pnpm audit` advisories in dev-only paths (vitest → esbuild, vitest → vite) — not in the runtime tree. The third (fast-xml-parser ≤4.5.0) is in the runtime path; tracked for v1.0.1 bump.
